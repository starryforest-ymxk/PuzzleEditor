# 任务报告 - 2025-12-13 条件类型下拉样式修复

## 问题
条件构造器在 Windows 环境下切换条件类型时，类型下拉框退化为系统默认灰底，仅显示箭头，文字与主题色块丢失，影响可读性。

## 变更
- 调整 `condition/conditionStyles.ts` 的 `typeSelectStyle`：
  - 显式设置 `appearance/WebkitAppearance/MozAppearance` 以屏蔽系统默认样式覆盖。
  - 固定背景色、文字色、边框色与行高，并增加最小宽度/高度与左右内边距，避免文字被压缩。
  - 去除背景图箭头，改为容器内自绘文字箭头（覆盖 appearance:none 后的空态），彻底规避 Windows 下重复平铺的花纹问题。

## 影响
- 条件类型下拉框现在在 Windows 下保持自定义主题配色，文字可见，避免“只有箭头”的退化展示。

## 验证
- 手动检查样式定义，确认已包含跨浏览器外观声明与明确的背景/文字颜色，并改为容器内自绘箭头。未运行自动化构建/预览，请在浏览器中快速回归查看条件类型下拉展示。
